1. Некоторые обозначения

  • Под переменной будем иметь в виду количество вершин.
  • Под переменной будем иметь в виду количество ребер.
  • Все вершины кодируются последовательно натуральными числами, начиная с нуля.
  • По умолчанию имеется в виду ориентированный граф, если не сказано иначе.

2. Хранение вершин

Пусть последовательность натуральных чисел кодирует вершины. Тогда можно хранить граф внутри массива, причем двумя способами:

  1. Матрица смежности:

    vector<vector<bool>> g(n, vector<bool>(n));
    g[2][4] = true; // добавление ребра 2 -> 4
    bool adj = g[1][3]; // запрос вида "есть ли ребро 1 -> 3?"

  2. Список смежности:

    vector<vector<int>> g(n);
    g[2].push_back(4); // добавление ребра 2 -> 4
    for (int u : g[4]) { // обход всех вершин, смежных с вершиной 4
      cout << u << "\n";
    }

  3. Список ребер:

    struct { int u, int w; } edge;
    vector<edge> edges(m);

От выбора способа хранения зависит асимптотическая сложность обхода всего графа (вершин и исходящих из них ребер).

  • Для матрицы смежности сложность составляет :

    vector<vector<bool>> g(n, vector<bool>(n));
    for (int v = 0; v < n; ++v) {
      for (int u = 0; u < n; ++u) {
        if (g[v][u]) {
          do_something(v, u);
        }
      }
    }

  • Для списка смежности сложность равна :

    for (int v = 0; v < n; ++v) {
      for (int u : g[v]) {
        if (g[v][u]) {
          do_something(v, u);
        }
      }
    }

    Обратим внимание на то, что второй цикл суммарно сделает итераций. При этом может быть, что , отсюда и сложность .

Список ребер полезен для некоторых алгоритмов (Беллмана-Форда, Краскала).

3. DFS

Depth-first search, обход в глубину.

Рекурсивно:

vector<vector<int>> g(n);
vector<bool> used(n); // used[v] - посещена ли вершина v?

void preorder(int v);
void postorder(int v);

void dfs(int v) {
  used[v] = true;
  preorder(v);
  for (int u : g[v]) {
    if (!used[u]) {
      dfs(u);
    }
  }
  postorder(v);
}

// main():

dfs(1); // запуск DFS из вершины 1

С помощью стека:

vector<vector<int>> g(n);
vector<bool> used(n);

// main():

stack<int> st;
st.push(1)

while (!st.empty()) {
  int v = st.top();
  st.pop()

  used[v] = true;
  do_something(v);
  for (int u : g[v]) {
    if (!used[u]) {
      st.push(u);
    }
  }
}

4. BFS

Breadth-first search, обход в ширину.

Алгоритм аналогичен DFS с помощью стека, но теперь вместо него используем очередь.

vector<vector<int>> g(n);
vector<bool> used(n);

// main():

queue<int> q;
q.push(1)
used[1] = true;

while (!q.empty()) {
  int v = q.front();
  q.pop()

  do_something(v);
  for (int u : g[v]) {
    if (!used[u]) {
      used[u] = true;
      q.push(u);
    }
  }
}

5. Топологическая сортировка

Пусть нам дан DAG (Directed Acyclic Graph, направленный ациклический граф), и мы хотим упорядочить вершины так, чтобы ребра вели из младших по порядку вершин к старшим.

Идея такая: будем поочередно добавлять самые «глубокие» вершины. Для каждой непосещенной вершины запустим DFS, где в postorder добавим в массив. В результате получим обратный порядок, поэтому массив надо перевернуть.

vector<vector<int>> g(n);
vector<bool> used(n);
vector<int> order;

void dfs(int v) {
  used[v] = true;
  for (int u : g[v]) {
    if (!used[u]) {
      dfs(u);
    }
  }
  order.push_back(u);
}

// main():

for (int v = 0; v < n; ++v) {
  if (!used[v]) {
    dfs(v);
  }
}
reverse(order.begin(), order.end());

6. Поиск компонент связности в неориентированном графе

Из каждой непосещенной вершины запускаем DFS/BFS, внутри которого помечаем вершины номером компоненты связности.

vector<vector<int>> g(n);
vector<bool> used(n);
int cur = 0; // номер текущей компоненты. В конце алгоритма равен их количеству
vector<int> comp(n, -1); // comp[v] - номер компоненты связности для вершины v

void dfs(int v) {
  used[v] = true;
  comp[v] = c;
  for (int u : g[v]) {
    if (!used[u]) {
      dfs(u);
    }
  }
}

// main():

for (int v = 0; i < n; ++v) {
  if (!used[v]) {
    dfs(v);
    ++c;
  }
}
reverse(order.begin(), order.end());

Можно обойтись без массива used, делая вместо него проверку вида if (comp[v] == -1).

7. Компоненты слабой связности

Компоненты слабой связности в ориентированном графе эквивалентны компонентам связности в неориентированном.

Алгоритм примерно такой:

  1. Делаем неориентированный граф, добавляя недостающие обратные ребра
  2. Ищем компоненты связности в полученном неориентированном графе

8. Компоненты сильной связности

  • Вершины и называются сильно связными, если из можно попасть в (любым возможным маршрутом в графе) и наоборот — из в .
  • Компонента сильной связности — множество попарно сильно связных вершин.

Если так подумать, то сильная связность между вершинами — это отношение эквивалентности, а компоненты сильной связности — классы данного отношения.

Тривиальный алгоритм выглядит так:

  1. Пусть — множество вершин, которые достижимы из в .
  2. С помощью любого обхода найдем для каждого .
  3. Для каждого найдем — множество вершин, которые достижимы из в
  4. Компонентой сильной связности для вершины будет являться .

Этот алгоритм работает за . Для каждой из вершин мы запускаем обходы за .

Однако можно найти компоненты сильной связности за :

  1. Делаем топологическую сортировку
  2. Строим транспонированный граф
  3. С помощью любого обхода запускаем DFS в порядке топсорта, внутри которого помечаем вершины номерами компоненты связности.

Реализация:

vector<vector<int>> g(n);
vector<vector<int>> t(n);
vector<bool> used(n); // используется для топсорта
vector<int> order; // порядок топсорта
int c = 0; // текущий номер компоненты
vector<int> comp(n, -1); // comp[v] - номер компоненты для вершины v

void dfs_topsort(int v) {
  used[v] = true;
  for (int u : g[v]) {
    if (!used[u]) {
      dfs(u);
    }
  }
  order.push_back(u);
}

void dfs_comp(int v) {
  comp[v] = c;
  for (int u : t[v]) { // <--- обходим транспонированный граф!
    if (comp[u] == -1) {
      dfs(u);
    }
  }
}

// main():

// 1. топсорт
for (int v = 0; v < n; ++v) {
  if (!used[v]) {
    dfs_topsort(v);
  }
}
reverse(order.begin(), order.end());

// 2. транспонирование
for (int v = 0; v < n; ++v) {
  for (int u : g[v]) {
    t[u].push_back(v); // обратите внимание на порядок u -> v
  }
}

// 3. помечание компонент связности
for (int v = 0; v < n; ++v) {
  if (comp[v] == -1) {
    dfs_comp(v);
    ++c;
  }
}

Объяснение того, почему этот алгоритм работает, можно найти в статье от USACO.

Существует алгоритм Тарьяна, который позволяет найти компоненты сильной связности всего за один проход DFS вместо двух. Рассматривать его не буду.

9. Конденсация графа

Суть в том, чтобы «сжать» целую компоненту слабой/сильной связности в одну вершину. Это и будет называться конденсацией графа.

Запустим любой обход. Если встречается ребро, переходящее проходящее между вершинами из разных компонент связности, то добавляем в конденсацию ребро между этими компонентами.

vector<vector<int>> g(n);
vector<bool> used(n);

// пусть мы уже нашли компоненты сильной/слабой связности
int cnt; // количество компонент
vector<int> comp; // comp[v] - номер компоненты для вершины v

// список смежности конденсации графа
// хотя правильнее сказать множество смежности; он тут будет удобнее
vector<unordered_set<int>> condensation(c);

void dfs(int v) {
  used[v] = true;
  for (int u : v) {
    if (comp[v] != comp[u]) {
      condensation[comp[v]].insert(comp[u]);
    }
    if (!used[u]) {
      dfs(u);
    }
  }
}

for (int v = 0; v < n; ++v) {
  if (!used[v]) {
    dfs(v);
  }
}

10. Поиск цикла в ориентированном графе

Введем три цвета: белый, серый, черный. Изначально все вершины белые. Для каждой белой вершины запускаем DFS. Внутри него, в preorder помечаем вершину серым, а в postorder черным. Если смежная вершина серая — цикл найден.

const int WHITE = 0, GRAY = 1, BLACK = 2;
vector<vector<int>> g(n);
vector<int> color(n, WHITE);
bool has_cycle = false;

void dfs(int v) {
  color[v] = GRAY;
  for (int u : g[v]) {
    if (color[u] == GRAY) {
      has_cycle = true;
    } else if (color[u] == WHITE) {
      dfs(u);
    }
  }
  color[v] = BLACK;
}

// main():

for (int v = 0; v < n; ++v) {
  if (color[v] == WHITE) {
    dfs(v);
  }
}

10.1. Восстановление ответа

Можно хранить массив предков и обновлять его в preorder. Когда нашли цикл, проходимся по этому массиву, пока не дойдем обратно до вершины .

11. Поиск эйлерова цикла

  • Эйлеров путь — путь, в котором каждое ребро встречается только один раз.
  • Эйлеров цикл — эйлеров путь, являющийся циклом :)

Теорема. Эйлеров цикл существует тогда и только тогда, когда:

  1. Степень каждой вершины четная
  2. Граф связный
vector<unordered_set<int>> g(n); // множество смежности для удобства
vector<int> path;

// алгоритм поиска эйлерова цикла
// работает корректно, если выполнены достаточные условия
void euler(int v) {
  while (!g[v].empty()) {
    int u = g[v][0];
    g[v].erase(u);
    // если граф неориентированный:
    // g[u].erase(v);
    euler(u);
  }
  path.push_back(v);
}


// main():

// 1-ое условие: deg(v) четный
for (int v = 0; v < n; ++v) {
  if (g[v].size() % 2 != 0) {
    cout << "impossible\n";
    return 0;
  }
}

// 2-ое условие: граф связный
if (!is_connected()) {
  cout << "impossible\n";
  return 0;
}

euler(0); // или от любой другой вершины

// теперь в path хранится эйлеров путь

13. Алгоритм Дейкстры

Алгоритм поиска кратчайшего пути от одной вершины до всех остальных во взвешенном графе. Важное условие — все ребра неотрицательного веса.

13.1. Реализация алгоритма

Первый вариант — линейно искать вершину с наименьшим . Работает за . Полезен .

def dijkstra(s):
  used = [False] * n
  d = [INF] * n
  d[s] = 0

  for i in range(n):
    v = -1
    for j in range(n):

Третий вариант — использовать приортетную очередь. Асимптотически такой же, как и с деревом поиска.

Если под рукой реализация приоритетной очереди, не поддерживающая операцию decrease_key (например, бинарная куча), то можно просто пропускать вершины с неактуальным расстоянием.

from heapq import heappush, heappop

def dijkstra(s):
  d = [INF] * n
  d[s] = 0

  q = []
  heappush(q, (0, s))

  while q:
    w_cur, v = heappop(q)
    if w_cur > d[v]:
      continue

    for u, w in g[v]:
      d[u] = min(d[u], d[v] + w)
      heappush(q, (d[u], u))

13.2. Инвариант

  • Если вершина не посещена, то — длина кратчайшего пути, полученного из посещенных вершин.
  • Если вершина посещена, то — окончательная длина кратчайшего пути.

Корректность алгоритма доказывается на данном инварианте как индукционном предположении.
